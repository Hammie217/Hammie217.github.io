EEE6225 Systems Design

Tutorial 4: GF(28) Exponentiation
1. GF(28) Parallel Multiplier

Building on the knowlege you have learned from designing a GF(24) multiplier, your task is to design and test a GF(28) multiplier for a primitive polynomial, P(x) = x8+x4+x3+x+1. If your design for tutorial 3 was written generically this should be relatively straight forward. Your result should be a VHDL description for a parallel GF(28) multiplier which you should test for correctness by writing a suitable VHDL testbench.

Your results should include the following:

  • VHDL description of your multiplier
  • VHDL testbench for validating the correct functioning of the multiplier
  • Performance results (area used and time taken)
2. Looped Exponentiation

Using the multiplier you have just designed now add a register and suitable control logic so it will receive an 8-bit value, x when instructed to do so by a "load" handshake signal and over a number of clock cycles use repeated multiplication to compute x254 and finally signal completion by asserting a "done" handshake signal.

Your results should include the following:

  • VHDL description of your looped exponentiator
  • VHDL testbench for validating the correct functioning
  • Performance results (area used and time taken)
3. Parallel Exponentiation

The process of calculating x254 in GF(28) can be done using only 4 off 2-input multipliers (AxB) and 3 trivial constant multipliers (eg 2xA, 4xA, 16xA).

Draw the circuit for this showing the multipliers and the resulting powers of x.  Using your multiplier synthesise this circuit, validate its correct operation and comment on the relative performance of the different methods of exponentiation.

Your results should include the following:

  • VHDL description of your (optimal) parallel exponentiator
  • VHDL testbench for validating the correct functioning
  • Performance results (area used and time taken)
  • Comments on relative performance of exponentiators
